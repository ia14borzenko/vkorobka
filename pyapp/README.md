# Vkorobka Python Test Application

Python приложение для тестирования компонентов системы vkorobka.

## Установка

```bash
pip install -r requirements.txt
```

## Использование

```bash
python main.py
```

### Графический интерфейс (все функции CLI)

Модульное приложение на **tkinter** (каталог `gui/`): одно подключение к win-x64, вкладки для сети, дисплея (RGB565), текстинга, микрофона, динамика и диагностики. Параметры соответствуют `voice_cli.py`, `speaker_cli.py`, `show_bg.py`, `texting_cli.py`, `main.py`.

Запуск **из каталога `pyapp`** (нужен запущенный win-x64, UDP 1236):

```bash
python gui_app.py
```

1. Вкладка **«Сеть»** — указать хост/порт, нажать **Подключить**.
2. Остальные вкладки вызывают те же действия, что и одноимённые CLI-скрипты; длительные операции выполняются в фоне, сообщения — в **«Журнал»** внизу окна.

### Динамик ESP32 (MAX98357A)

После запуска **win-x64** и подключения ESP32 по Wi‑Fi/TCP:

```bash
python speaker_cli.py --host 127.0.0.1 --port 1236 path/to/track.wav
```

Моно/стерео (стерео усредняется); частота приводится к **22050 Hz**, **16-bit** через **scipy**. В сеть уходит **MSG_TYPE_STREAM**, `stream_id=3`, моно PCM **16-bit LE** (легче, чем 48 kHz / 24-bit). Перед потоком — **`dyn.on`**, после — **`dyn.off`**.

## Описание

Приложение последовательно тестирует два основных компонента системы:
1. win-x64 - локальный компонент
2. esp32 - через win-x64

Для каждого компонента отправляется тестовая JPG картинка (4 КБ), компонент должен отзеркалить её и вернуть обратно.

## Требования

- Python 3.8+
- Pillow для работы с изображениями
- UDP порт 1236 должен быть доступен для связи с win-x64 приложением
